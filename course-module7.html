Explain Like You're 5

Unit tests are like checking each LEGO piece before building the castle. If one brick is cracked, you fix it before stacking everything else.

Arrange

Setup dependencies, fakes, and input data.

Act

Call the method you're testing.

Assert

Verify outputs with expect matchers.

import 'package:flutter_test/flutter_test.dart'; import 'package:price_calculator/price_service.dart'; void main() { group('PriceService', () { test('applies weekend discount', () { final service = PriceService(); final total = service.total( subtotal: 100, isWeekend: true, ); expect(total, equals(90)); }); test('throws when subtotal negative', () { final service = PriceService(); expect( () => service.total(subtotal: -5, isWeekend: false), throwsA(isA()), ); }); }); }
Tip: Keep unit tests in test/, mirror the lib structure, and run with flutter test in under 5 seconds for fast feedback.

Lesson 7.2 ¬∑ Widget Testing

WidgetTester APIs, pumps, finders, semantics, golden tests
üé®

Explain Like You're 5

Widget tests are like dress rehearsals on a miniature stage. You simulate taps, scrolls, and check what appears without launching the full app.

import 'package:flutter_test/flutter_test.dart'; import 'package:todo_app/todo_list.dart'; void main() { testWidgets('adds a todo item', (tester) async { await tester.pumpWidget(const TodoApp()); await tester.enterText(find.byType(TextField), 'Buy milk'); await tester.tap(find.text('Add')); await tester.pump(); // rebuild with new state expect(find.text('Buy milk'), findsOneWidget); }); testWidgets('checkbox toggles', (tester) async { await tester.pumpWidget(const TodoApp()); final checkbox = find.byType(Checkbox).first; await tester.tap(checkbox); await tester.pumpAndSettle(); expect( tester.widget(checkbox).value, isTrue, ); }); }

Explain Like You're 5

Integration tests are like taking the whole robot outside and seeing if it can deliver a package. You test everything together, not just individual gears.

import 'package:integration_test/integration_test.dart'; import 'package:flutter_test/flutter_test.dart'; import 'package:shop_app/main.dart' as app; void main() { IntegrationTestWidgetsFlutterBinding.ensureInitialized(); testWidgets('checkout flow works', (tester) async { app.main(); await tester.pumpAndSettle(); await tester.tap(find.text('Add to cart').first); await tester.tap(find.byIcon(Icons.shopping_cart)); await tester.tap(find.text('Checkout')); await tester.pumpAndSettle(); expect(find.text('Order confirmed'), findsOneWidget); }); }
Layer Tool Runtime Purpose
Logic flutter test Milliseconds Pure Dart unit tests
UI flutter test ~1 sec/widget Widget tests headless
End-to-End integration_test Minutes Full app on device/emulator
Mocking Cheat: Use Mocktail or Mockito to stub repositories. Prefer fakes (in-memory implementations) when behavior is simple.

üéØ Interview Drill

1. Difference between widget and integration tests?

Widget tests run inside Flutter's test harness without real devices; integration tests spin up the actual app on a device/emulator.

2. How do you mock HTTP requests?

Inject a custom http.Client, use MockClient from http/testing.dart or Mocktail to intercept calls.

3. Why use golden tests?

They detect visual regressions by diffing screenshots pixel-by-pixel.

4. What is pumpAndSettle?

It keeps pumping frames until the Widget tree is idle‚Äîhandy after animations or navigation.

5. How to run integration tests in CI?

Use flutter test integration_test with --device-id or flutter drive, ensure emulators/simulators are booted.
